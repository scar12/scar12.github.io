Github Pages + hexo 搭建个人博客

发表于 2020-03-09 更新于 2020-03-10 阅读次数：

前言

这是本人写的第一篇博客。搭建这个博客是为了记录下自己学习过程中遇到与解决的问题，算是一种经验的积累。另一方面，通过对已学知识（技术）进行总结，提升自己的文字表达能力以及建立知识体系。初衷不是为了写给别人看的，所以也不担心别人认为没有价值。不过如果能帮助别人解决问题的话，那就更好了。

下面是众多博主分享的经验结合本人实际的建站流程。由于是在Windows环境下搭建，其他平台用户搭建过程细节会有所不同。

环境搭建

Hexo 简介

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。利用静态模板系统，即使没有网页前端开发经验，我们也能一定程度上个性化自己的博客页面。Github 官方推荐的是 Jekll，但我觉得搭建 jekll 需要的依赖环境比较复杂（ruby + python2），加上主题风格不太喜欢，所以改用 Hexo。当然 jekll 也有它的优点。

安装 Node.js

Hexo 是基于 Node.js 编写的，所以需要安装 Node.js 和里面的 npm 工具。

Node.js 为大多数平台提供了官方的安装程序。下载慢的话，可以尝试淘宝 Node.js 镜像。安装时保持默认设置即可。

安装程序完成后，可以打开命令提示符检查是否成功安装：输入 node -v、npm -v，出现版本号则说明成功安装。

![image-20200309152543140](Github Pages + hexo 搭建个人博客/image-20200309152543140.png)

安装 Git

Git 是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。也就是说，它是我们用来管理 hexo 博客文章，上传到 GitHub 的工具。

到官网下载安装即可。

![image-20200309023514325](Github Pages + hexo 搭建个人博客/image-20200309023514325.png)

安装完成后，桌面右键，点击 Git Bush Here，输入 git --version，出现版本号则说明 Git 安装成功。

![image-20200309023810499](Github Pages + hexo 搭建个人博客/image-20200309023810499.png)

The BASH emulation behaves just like the “git” command in LINUX and UNIX environments.

在Windows 上，通过 Git-Bash 模拟 LINUX 和 UNIX 环境下的 “git” 命令。
