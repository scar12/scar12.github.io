Git 常用命令

本文整理了一些Git的命令及其用法，后续会继续更新。

目前主要的参考资料为廖雪峰大牛的Git教程，之后会参考其他资料进行补充。

基本命令

创建

$ mkdir <folder> 创建名为<folder>的文件夹。

$ pwd 获取当前目录

$ git init 初始化，把当前目录变成 Git 可管理的仓库。

添加和提交

工作区和暂存区

$ git add <file> 把文件从工作区添加到暂存区，可多次add不同文件。

$ git commit -m <message> 把文件从暂存区提交到版本库（master）。可一次提交多个文件。<message> 是本次提交的说明。

$ git status 查看工作区和暂存区当前状态，是否有未 add 或 commit 的修改操作。

注意，modified的字体颜色：

未add（changes not staged for commit，需要提交的修改未添加到暂存区）为红色，

已add未commit（change to be commit，需要提交的修改已添加到暂存区）为绿色。

$ git diff <file> 查看修改内容（文件的工作区版本与暂存区版本区别）。

上图git add后版本同步。

版本回退

$ git log 查看提交日志。

$ git log --pretty=oneline 只显示<commit-id>（版本号）以及说明。

$ git reset --hard HEAD^ 回退到上一个版本，回退N个版本写成HEAD~N。

$ git reflog 查看所有命令记录。可以借此获取log命令无法查看的已回退的”新“版本号，通过$ git reset --hard <commit-id>来恢复到“新”版本。
